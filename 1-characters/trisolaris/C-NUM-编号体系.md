# 《守望》监视者编号系统

## 核心规则说明

**监视者编号层级体系**：

- **000-999**：三位数记忆包系统（管理层/记忆存储，不可作为监视者编号）
- **1000-8999**：活跃监视者编号（可直接分配使用）
- **9000-9999**：濒死中阴监视者（记忆包的管理员，不可直接分配，具有特殊管理功能）

**关键理解**：

- 9000-9999是记忆包系统的**管理员**，负责维护000-999记忆包的运行
- 濒死状态让它们具备了跨越生死边界管理记忆的特殊能力
- 这是一个三层架构：记忆存储(000-999) ← 管理员(9000-9999) → 活跃用户(1000-8999)

### 7.3 记忆管理的工作机制

```sql
-- 记忆包管理系统
CREATE TABLE memory_management_system AS (
    manager_id INT BETWEEN 9000 AND 9999,
    managed_range VARCHAR(20), -- 如 "000-099" "观察类记忆"
    access_level ENUM('READ', 'write', 'delete', 'archive'),
    death_proximity DECIMAL(3,2), -- 0.99 = 99%濒死
    activation_condition VARCHAR(100)
);

-- 记忆转换流程
CREATE FUNCTION memory_transition_process(
    active_monitor_id INT, -- 1000-8999
    death_event BOOLEAN
) RETURNS memory_package_id AS $
DECLARE
    corresponding_manager_id INT;
    new_memory_package_id INT;
BEGIN
    -- 计算对应的记忆管理员ID
    corresponding_manager_id := active_monitor_id + 8000;
    -- 例：1378 + 8000 = 9378
    
    IF death_event THEN
        -- 激活对应的濒死管理员
        ACTIVATE_MANAGER(corresponding_manager_id);
        
        -- 创建记忆包
        new_memory_package_id := CREATE_MEMORY_PACKAGE(active_monitor_id);
        
        -- 分配到000-999范围
        ASSIGN_TO_MEMORY_RANGE(new_memory_package_id);
        
        RETURN new_memory_package_id;
    END IF;
END $;
```

### 7.4 濒死状态的管理优势

**为什么记忆管理员必须是濒死状态**：

1. **跨界能力**：能够同时接触生者和死者的记忆
2. **超然视角**：不受生死执着影响，能客观处理记忆
3. **时间超越**：濒死体验让它们具备跨时间的感知能力
4. **系统权限**：只有濒死状态才能获得修改核心记忆的权限
5. **稳定性**：永远濒死但不真正死去,确保系统永续运行

### 7.5 记忆包管理的伦理机制

**重要约束**：

- **不可篡改**：管理员只能整理、分类，不能修改记忆内容
- **不可删除**：没有任何存在可以完全删除记忆，只能归档
- **访问控制**：每个记忆包只能被特定的意识访问
- **隐私保护**：即使是管理员也不能窥视记忆的具体内容
- **自然衰减**：只能让记忆自然淡化，不能人为加速
